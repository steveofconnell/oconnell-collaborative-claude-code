# Figure and Table Notes

Write a detailed, self-contained note for a figure or table so that a reader who skipped straight to it can understand it completely without consulting the body text.

## Step 1: Identify the Target

Determine which figure or table needs notes. In order of priority:
1. If the user specifies a file path or figure/table number, use that.
2. If the conversation has been discussing a specific figure or table, use that.
3. If ambiguous, ask.

## Step 2: Read the Source

- **For figures:** Read the manuscript .tex file containing the `\includegraphics` call. Also open the figure file itself (PDF/PNG) to see what is actually depicted.
- **For tables:** Read the .tex table output file (in `5manuscript/tables/` or equivalent). Then find the script that generates it — grep for the output filename across `4code/`, `2processing/`, and any other code directories.

If the table already has notes, read them — you may be updating, not writing from scratch.

## Step 3: Gather Context

Before writing the note, collect what you need:
- Read the relevant section of the manuscript for variable definitions, sample descriptions, and specification details.
- If it is a regression table, identify: the estimand, fixed effects, SE type, inference method, control variables, sample restrictions.
- If it is a descriptive table or figure, identify: unit of observation, sample, data source, time period, any constructed variables.
- If colors, line types, shading, or markers are used in a figure, catalog all of them.

Do not write the note until you have this information. If something is missing and cannot be inferred, ask the user.

## Step 4: Write the Note

The note must include every applicable component from this checklist:

### Required for all tables and figures
1. **What is shown.** One sentence stating what the table/figure displays.
2. **Reading example.** A concrete plain-language gloss of one specific value: "A value of 0.26 indicates that the county's crop-weighted average market price was 26 percent below parity." This is mandatory — every note gets at least one.
3. **Sample and restrictions.** N, unit of observation, any subsetting or exclusion criteria.
4. **Data source.** Where the underlying data come from.

### Required for tables
5. **Column/row decoder.** If the table has grouped columns, panel rows, or non-obvious structure, explain what each group represents.
6. **Variable definitions.** Every non-obvious variable defined inline, with formula if computed.
7. **Statistical details.** SE type (and how displayed — parentheses, brackets), inference method, significance stars key. If the inference method is non-standard (e.g., sharpened FDR q-values rather than p-values), say so explicitly.
8. **Fixed effects and controls.** List them.
9. **Anomalies.** Missing cells, dropped periods, unusual column counts — explain why.

### Required for figures
10. **Visual decoder.** What every color, line type, shading level, marker, and annotation means. The reader should not need a legend that is hard to read at print size.
11. **Substantive reading observation.** One sentence directing the reader to the key pattern the figure is meant to show. (For maps: what the geographic pattern reveals. For time series: what the temporal pattern shows.)

### Always (as a LaTeX comment, not in the visible note)
12. **Script provenance.** `% Source: 4code/03_regressions.R` or equivalent.

### Style
- One dense paragraph. Not bulleted, not multi-paragraph. 3–8 sentences depending on complexity.
- Begin with `\textit{Notes:}` (for minipage/tablenotes patterns) or embed in `\caption{}` (for figure captions that carry the full note).
- Compact but complete. Every word earns its place.

## Step 5: Place the Note

### Figures (note goes in manuscript .tex)

Insert the note in the manuscript file containing the `\includegraphics` call, inside the `figure` environment. Two patterns depending on context:

**If the caption is short (just a title):** Expand the `\caption{}` to include the full note — variable definition, visual decoder, reading observation, source. This is the preferred pattern for figures.

**If a separate note block is needed** (e.g., the caption is already long, or house style separates them): Add a `\minipage` block after `\caption` and before `\end{figure}`:
```latex
\begin{minipage}{\linewidth}
\vspace{0.5em}
\footnotesize
\textit{Notes:} [note text]
\end{minipage}
```

Add a `% Source:` comment above or below the figure environment.

### Tables (note goes in the generating script)

**HARD RULE: Never edit a .tex file that is produced by a script.** If a table .tex file is generated by R, Python, or Stata code, the note must be added to the *script* so that re-running the script reproduces the table with notes intact. Editing the .tex output directly will be silently overwritten on the next run and the note will be lost. If the script's output framework (e.g., stargazer) does not support rich notes natively, add a post-processing step in the same script that injects the note into the .tex file after the table is written — this is still part of the pipeline. The note must travel with the code, not live only in the output.

The exact insertion depends on the output method:

**R with `stargazer`, `modelsummary`, `kableExtra`, `xtable`, or similar:**
- Find the function call that writes the table .tex file.
- Add the note as a `notes` argument, `add_footnote()`, `footnote()`, or post-processing step — whichever the package supports.
- If the package does not support notes natively, append the minipage block to the output using `cat()` / `writeLines()` after the table is written, or use `add_rows()` to insert a `\multicolumn` note row.

**R with `threeparttable` output:**
- Insert via `\begin{tablenotes}[para] \item \textit{Notes:} [text] \end{tablenotes}` inside the threeparttable environment.

**Python with `stargazer`, `pandas.to_latex()`, or similar:**
- Append the minipage block to the LaTeX string before writing to file.

**Stata with `esttab`, `outreg2`, `estout`:**
- Use the `postfoot()` or `addnotes()` option to embed the note.

**If the table .tex is written manually (e.g., via `cat()` line by line):**
- Insert the note rows directly in the appropriate place in the output code.

After modifying the script, run it to regenerate the table and verify the note appears correctly. Use `/gen` if convenient.

### Special case: table .tex exists but no generating script is found

If the table is hand-written or the script cannot be located, insert the note directly into the .tex file using the minipage or tablenotes pattern, and flag to the user that future regeneration of the table will overwrite the note.

## Step 6: Verify

Read the output .tex file (for tables, after regeneration) and confirm:
- The note renders valid LaTeX (no unescaped characters, balanced braces).
- All checklist items that apply are present.
- The reading example uses a real value from the table/figure, not a placeholder.

## Self-Containment Test

Before finishing, ask: "Could a reader who has never seen the body text of this paper understand this table/figure completely from the caption and notes alone?" If no, something is missing — go back to Step 4.
